kvviz — Live Fragmentation Dashboard

Real-time block-level KV cache fragmentation visualization

Total Blocks iTotal size of the block pool. Grows dynamically as requests need more memory (unless Max Blocks is set).
—
Free Blocks iBlocks not currently owned by any request. Available for new allocations.
—
Used Blocks iBlocks currently allocated to active requests holding KV cache data.
—
Active Requests iNumber of requests currently generating tokens. Each request holds blocks until it finishes.
—
Fragmentation i1 - (largest contiguous free run / total free blocks). 0% = all free blocks are together (perfect). 90% = free blocks are scattered into tiny gaps. High fragmentation means new large allocations can fail even when total free space is sufficient.
—
CUDA Frag iGPU-level memory fragmentation from PyTorch's CUDA allocator: (reserved - active) / reserved. Only populated when connected to a real GPU workload (vLLM mode). Empty in synthetic mode.
—
Block Occupancy Grid iEach square is one memory block. Colored = owned by a request (color = request identity). Dark = free. Watch fragmentation happen as colored blocks scatter and gaps appear between them. Hover a cell to see its block index and owner.
iFragmentation ratio over time (0 to 1). Measures how scattered the free blocks are. Computed as 1 - (largest contiguous free run / total free blocks). Spikes when requests free non-adjacent blocks, drops when large contiguous regions open up.
iStacked area showing used blocks (orange) vs free blocks (green) over time. The total height is the pool size. Watch it ramp up as requests arrive, dip when requests finish and free blocks, then ramp again.
iHow efficiently tokens fill their allocated blocks (0 to 1). A request using 50 tokens in 4 blocks of 16 = 50/64 = 0.78. The remaining 14 tokens of capacity are "tail waste". Smaller block sizes improve packing but increase overhead.
iReal GPU memory from PyTorch's CUDA allocator. Active (blue) = memory in use. Inactive split (red) = fragmented blocks the allocator reserved but can't use. Reserved (dashed) = total GPU memory claimed. Only populated in vLLM live mode.
